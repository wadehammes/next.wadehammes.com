# Prismic

This chapter covers how Prismic content reaches the site: codegen, the client, getters, parsers, preview, and Slice Simulator.

## Generated types

Types are generated by **`prismic-ts-codegen`** into [src/prismic/types/prismic.generated.ts](../../src/prismic/types/prismic.generated.ts).

### Regenerating types

```bash
pnpm types:prismic
```

Requires env vars (see [platform.md](platform.md)). Configuration lives in:

- [prismicCodegen.config.ts](../../prismicCodegen.config.ts) — output path, remote vs local models.
- [prismic.config.json](../../prismic.config.json) — Slice Machine repo metadata.

**Node 24+**: The published `prismic-ts-codegen` CLI can crash on newer Node; this repo runs a local wrapper at [scripts/prismic-codegen.cjs](../../scripts/prismic-codegen.cjs) that mirrors the CLI steps. Prefer `pnpm types:prismic` over `pnpm types:prismic:cli` until upstream is fixed.

When both `PRISMIC_CUSTOM_TYPES_API_TOKEN` and `PRISMIC_REPOSITORY_NAME` are set, models are fetched from the Custom Types API. Optional local overrides can live under `prismicio/customtypes/**` or `prismicio/slices/**`.

**Never hand-edit** `prismic.generated.ts`.

## Client

[src/prismic/client.ts](../../src/prismic/client.ts) exports `getPrismicClient()`:

- Singleton `createClient<AllDocumentTypes>` bound to the repository name from env.
- Uses `PRISMIC_ACCESS_TOKEN` when the repository is private.
- Calls `enableAutoPreviews({ client })` so draft mode requests resolve preview refs automatically.

## Constants and configuration guards

[src/prismic/constants.ts](../../src/prismic/constants.ts):

| Export | Purpose |
|--------|---------|
| `PRISMIC_DEFAULT_REVALIDATE_SECONDS` | ISR default (7 days). **`src/app/page.tsx` must duplicate this literal** in `export const revalidate`—Next.js 16 only accepts statically inlined segment config there. |
| `getPrismicHomeDocumentType()` | Custom type API ID for `getSingle` (default `"home"`, overridable via env). |
| `isPrismicConfigured()` | Whether repository env is present; getters return `null` when false. |
| `getPrismicRepositoryName()` | Repository slug for `PrismicPreview` toolbar in layout. |

## Getters

[src/prismic/getPage.ts](../../src/prismic/getPage.ts):

- **`getHomePage(options?)`** — Fetches the home singleton via `client.getSingle(type)`. Catches `NotFoundError` and returns `null`.
- **`getCachedHomePage()`** — React `cache()` wrapper for deduplication within a request.

Add new getters here (or sibling files) when you introduce additional document types or routes.

## Parsers

Parsers translate Prismic documents into app-level types used by components.

| File | Output | Notes |
|------|--------|-------|
| [parsePage.ts](../../src/prismic/parsePage.ts) | `ParsedPage` | Hero slice Rich Text, SEO fields, uid/id/lang. |
| [parseSeoMeta.ts](../../src/prismic/parseSeoMeta.ts) | `ParsedSeoMeta` | Trims and nulls empty `meta_title` / `meta_description`. |
| [parseImage.ts](../../src/prismic/parseImage.ts) | `ParsedImage` | Imgix URL and dimensions from image fields. |

**Pattern**: Use `prismic.isFilled.*` guards; return `null` for empty fields rather than empty strings in parsed output (except where components expect empty string fallbacks).

### Hero slice

`parseHomeDocument` reads the first filled `hero_section` slice's `primary.copy` field. If you add slices or change the home document shape, update the parser and regenerate types.

## Rich Text in components

Use `@prismicio/react`:

```tsx
import { PrismicRichText } from "@prismicio/react";
import * as prismic from "@prismicio/client";

if (!prismic.isFilled.richText(copy)) {
  return null;
}

return <PrismicRichText field={copy} />;
```

See [Bio.component.tsx](../../src/components/Bio/Bio.component.tsx).

## Preview and draft mode

### Route Handlers

| Route | Handler | Purpose |
|-------|---------|---------|
| `/api/preview` | [preview/route.ts](../../src/app/api/preview/route.ts) | `redirectToPreviewURL` with a `linkResolver` mapping `home` → `/`. |
| `/api/exit-preview` | [exit-preview/route.ts](../../src/app/api/exit-preview/route.ts) | `exitPreview()` from `@prismicio/next`. |

In Prismic → Settings → Previews, set the preview URL to `https://<your-domain>/api/preview`.

### Layout integration

[src/app/layout.tsx](../../src/app/layout.tsx):

- Reads `draftMode().isEnabled` and renders `PreviewModeOverlay` when active.
- Wraps the app in `PrismicPreview` when the repository name is configured.

Preview API routes use `Cache-Control: private, no-store` headers defined in [next.config.ts](../../next.config.ts).

## Slice Simulator

[src/app/slice-simulator/](../../src/app/slice-simulator/) hosts a local preview page for Slice Machine.

- Relaxed **`frame-ancestors`** CSP in `next.config.ts` allows Prismic and localhost to iframe `/slice-simulator`.
- Dev server runs on port **4431**; Slice Machine often uses **9999**—both are listed in allowed frame ancestors.

When adding slices, wire them in the simulator client components and ensure models are reflected in codegen.

## Metadata from Prismic

[src/app/page.tsx](../../src/app/page.tsx) `generateMetadata` reads `metaTitle` and `metaDescription` from `ParsedPage`, falling back to hard-coded site defaults when Prismic is unset or fields are empty.

When you add SEO fields or new document types, extend parsers and metadata generation together.
